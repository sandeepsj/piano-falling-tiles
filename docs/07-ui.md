# 07 — UI Layer

## Purpose

All Jetpack Compose screens and components. Provides navigation between screens,
wraps the `GLSurfaceView` game canvas, shows score/combo HUD, playback controls,
file picker, MIDI device picker, and settings.

Does NOT: contain game logic, audio code, or MIDI processing. It only reads
state from `ViewModel`s (which wrap the layers below) and dispatches user actions.

---

## Dependencies

- Layer 2 (Rendering) — `GameSurfaceView` embedded via `AndroidView`.
- Layer 1 (project setup) — Compose dependencies.
- No audio or MIDI layers directly (accessed through ViewModel).

## Tasks

### 7.1 Create `AppNavigation.kt` — NavHost with 4 screens
- **What**:
  ```kotlin
  sealed class Screen(val route: String) {
      object Home : Screen("home")
      object Game : Screen("game")
      object Results : Screen("results")
      object Settings : Screen("settings")
  }

  @Composable
  fun AppNavigation() {
      val navController = rememberNavController()
      NavHost(navController, startDestination = Screen.Home.route) {
          composable(Screen.Home.route) { HomeScreen(navController) }
          composable(Screen.Game.route) { GameScreen(navController) }
          composable(Screen.Results.route) { ResultsScreen(navController) }
          composable(Screen.Settings.route) { SettingsScreen(navController) }
      }
  }
  ```
- **How to verify**: App launches, shows HomeScreen. No crash.
- **Done when**: All 4 routes defined, `MainActivity` uses `AppNavigation()`.

## Integration Points

`GameViewModel` is the bridge — UI reads from it, game layers write to it:

```kotlin
// UI reads:
val gameState: StateFlow<GameState>
val isPlaying: StateFlow<Boolean>
val loadingProgress: StateFlow<Float>

// UI writes:
fun togglePlayPause()
fun seekTo(progress: Float)
fun setSpeed(Float)
fun setPracticeMode(Boolean)
fun loadSong(ByteArray)
```

`GameScreen` hosts `GameSurfaceView` via `AndroidView` — the GL canvas is below all Compose overlays.
